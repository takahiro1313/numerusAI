## 1. システム構成

```
┌──────────────────────────────────────────────┐
│ 参加者ブラウザ（配布URL）                       │
│  Streamlit UI                                 │
│   ├─ 🔒 Step1 データプレビュー                  │
│   ├─ 🟢 Step2 モデル選び（radio/slider）        │
│   ├─ 🟢 Step3 fit 編集セル → [学習]            │
│   └─ 🟢 Step4 predict 編集セル → [推論]         │
└───────────────────┬──────────────────────────┘
                    │ 「学習/推論」押下（編集セルの文字列を送信）
                    ▼
   ┌───────────────────────────────────────┐
   │ Streamlit サーバ（Community Cloud）    │
   │  - ダミーCSV読込→ラグ特徴量→時間分割     │
   │  - 限定名前空間で exec（fit/predict）   │
   │  - XGBoost/線形 学習・推論（数秒）      │
   │  - train/valスコア・重要度算出          │
   │  - 全担当予測を data/予測結果.csv に保存 ─┼─▶ ③まーぴーが読む
   └───────────────────────────────────────┘

  ※ 計算はサーバ側のみ（役員PCに負荷なし）。通しテーマ②「AIがデータからルールを学ぶ」。
  ※ 学習負荷分散のため 1人1URL（計5）＋予備1（デプロイ運用メモ）。
```

### 1.1 構成要素

| 要素 | 役割 | 技術 |
|---|---|---|
| Streamlit サーバ（Cloud） | 画面提供／データ管理／編集コードの安全実行 | Python, Streamlit |
| 数値AIモデル | 先行活動→翌月成約 の学習・推論 | XGBoost（`XGBRegressor`）／線形回帰（`LinearRegression`） |
| 編集セル | 参加者が `fit`/`predict` を書くUI | `st.text_area` |
| モデル/パラメータUI | 種類・`n_estimators`/`max_depth` | `st.radio` / `st.slider` |
| 出力 | 予測値・train/valスコア対比・重要度 | `st.metric` / `st.bar_chart` |

### 1.2 設計の要点

- **書く体験を fit/predict に絞る**：編集セルは2箇所。データ下ごしらえ・スコア・グラフ・CSV保存は 🔒。
- **IT①との対比**：IT＝「人が if を書く」→ AI＝「AIがデータからルールを学ぶ（fit）」。
- **過学習を体感**：時間分割（過去で学習→未来で検証）。Step2の数字を上げると訓練↑・検証↓。
- **Cloud実行**：XGBoost学習を役員PCで回さない。編集セルの文字列はサーバ側で `exec`。

### 3.1 入力→処理→出力

| 段 | 内容 | ゾーン |

| 入力 | 担当×月の時系列プレビュー | 🔒 見るだけ（Step1） |

### 3.2 編集ゾーン（🔒／🟢）

| 記号 | 該当 |
|---|---|
| 🔒 | データ読込・ラグ特徴量・時間分割・スコア計算・重要度グラフ・予測CSV保存・ボタン制御 |
| 🟢 | `fit`/`predict` の編集セル、モデル種類(radio)、パラメータ(slider)、担当(selectbox) |

- 画面冒頭に**やること一覧表**（Step1〜4・🔒/🟢）を掲示。

### 3.3 編集セルの初期スニペット（演習用／正解用）

**演習用（`app_ensyu.py`）— 空欄は `___`：**
```python
# 🟢 ===== ここを編集 =====   （学習 fit）
model.fit(___, ___)        # ← X_train, y_train を入れる
```
```python
# 🟢 ===== ここを編集 =====   （推論 predict）
pred = model.predict(___)  # ← X_new を入れる
```
**正解用（`app_seikai.py`）— 記入済み：** `model.fit(X_train, y_train)` ／ `pred = model.predict(X_new)`

> 🔒で用意：`build_model()`（radio/sliderからモデル生成）、`X_train/y_train/X_val/y_val/X_new`、スコア、重要度、`save_all_predictions`。

### 3.4 モデル選び・過学習（Step2・🟢）

- `st.radio` で **木のモデル（XGBoost）／直線モデル（線形回帰）** を選ぶ。
- 木のとき `n_estimators`(10–500)・`max_depth`(1–10) のスライダー。既定 `40/2`＝バランス◎、`300/6`＝過学習がはっきり。
- 重要度は木モデルのみ（`feature_importances_`）。

## 6. 安全な実行（exec）設計

参加者が書く `fit`/`predict` をサーバ側で実行するため、事故防止を徹底する。

| 項目 | 方針 |
|---|---|
| 名前空間の限定 | `exec(code, {"__builtins__": SAFE_BUILTINS}, ns)`。`ns` は `model`/`X_train`/`y_train`/`X_new` 等のみ |
| I/O 遮断 | `import`/ファイル/ネットワークI/Oは渡さない |
| 例外捕捉 | `try/except` で全例外を捕捉し**画面表示**（アプリは落とさない） |
| 前提 | 信頼できる少人数＋インスタンス分離（1人1URL）。ただし上記は必須 |

---

## 7. セキュリティ設計

| 項目 | 方針 |
|---|---|
| 外部通信 | なし（API・AIモデル呼び出しなし） |
| APIキー | 不要 |
| コード実行 | 限定名前空間で `exec`。不要な組み込み・I/Oを渡さない。例外は画面表示 |
| インスタンス分離 | 1人1URL（計5）。1台落ちても他へ影響なし |
| データ | **ダミーCSVのみ**。本番SFDC/TSR等は扱わない |

---

## 8. エラー処理

| 事象 | 挙動 |
|---|---|
| fit/predict の記述ミス | `_exec_user` が例外を捕捉→画面表示。`app_seikai.py` で答え合わせ |
| 未学習で推論 | 「先に学習してください」 |
| pred が未設定 | 「pred に予測結果が入っていません」 |
| 数字を極端に上げる | 学習は実行し、過学習として train/val 差に表れる（警告） |
| CSV未配置 | `python make_dummy.py` で生成 |
