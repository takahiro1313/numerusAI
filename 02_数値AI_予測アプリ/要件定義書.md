## 2. 目的・ゴール

「**数字を入れたら数字が出る**」数値AIを、参加者が **fit（学習）／ predict（推論）を自分で書いて** 動かす。さらに**パラメータを変えて反復**し、**過学習**を体感する。

- 題材：**営業の先行活動 → 翌月の成約数を予測**（フォーキャスト）。「来月どれくらい取れそう？」に答える。
- 特徴量：先行活動（問い合わせ・架電・面談・案件提案）の**当月・1ヶ月前・2ヶ月前**。**契約数（結果系）は使わない**（自明回避）。
- モデル：**XGBoost**（テーブルデータ・短時間学習。スライド50に明記）。
- 入力→処理→出力の型を、コードで一気通貫に通す原体験。
- 副次のねらい：重要度を見ると「**架電（量）より、提案・面談が翌月の成約に効く**」という示唆が得られる（行動につながる）。

## 4. 題材・データ（SFDCイメージのダミー時系列）

- ストーリー：**営業の先行活動 → 翌月の成約数を予測**（来月の見込みを読む）。
- データは **ダミーCSV**（本番SFDCの形を模す。実データは使わない）。**営業担当 × 月の時系列パネル**。
- 確定した活動量（ユーザー決定 2026-06-19）。**先行活動のみ**を特徴量に使い、契約数・成約数は特徴量に入れない。

### 4.1 生データのスキーマ（ダミーCSV）

| カラム | 例 | 役割 |
|---|---|---|
| `営業担当` | 担当01 | キー（パネルの単位） |
| `月` | 1〜30 | 時間 |
| `問い合わせ数` | 45 | 先行活動（特徴量の素） |
| `架電数` | 85 | 先行活動（※あえて効きにくい指標） |
| `面談数` | 28 | 先行活動 |
| `案件提案数` | 9 | 先行活動（最も効く） |
| `成約数` | 5 | その月の成約（→ 翌月にズラして正解 y にする） |

- 規模：**30担当 × 30ヶ月 ＝ 900行**（学習が数秒で終わる軽さ）。
- **特徴量づくり（🔒前処理）**：各活動の **当月・1ヶ月前・2ヶ月前** を作り（4活動×3＝**12特徴量**）、
  正解 y は **翌月の成約数**（`成約数` を1ヶ月先にずらす）。
- 目的変数は **翌月の成約数**（ユーザー確定。`Ad数` は廃止）。

## 5. 当日の進め方（スライド53：これからやること）

今回は**コードを書く**。fit → predict → パラメータ調整を体験。

| Step | 内容 | やること |
|---|---|---|
| Step1 | データを見る | 担当×月の時系列と成約を確認 → 入力の形を知る（🔒 見るだけ） |
| Step2 | 学習する | `model.fit(X, y)` を書く → [学習ボタン]（🟢） |
| Step3 | 推論する | **営業担当を選び**、`model.predict(X_new)` を書く → [推論ボタン]（🟢）→ その担当の翌月予測 |
| Step4 | パラメータを変える | 回数等を変えて何度も回す（過学習に注意）（🟢） |

- **検証は時間分割**：序盤の月で学習し、終盤の月（未来）で検証。「未来のデータを使わない」を体感。

### 5.1 推論エンドポイント（PdM FB 2026-06-19・3パート連結）

数値AIの出力を**生成AI（まーぴー）が話す**ための連結を用意する。

- **学習**：これまで通り **Streamlit アプリ上**（デプロイ済アプリ内）で行う。
- **推論**：まーぴーがこれを使い、**翌月の予測成約数**を音声で答える。**当日方針＝ベター（CSV読み）を確実に**。
  - **ベスト**：まーぴーが**推論エンドポイントを直接呼ぶ**（リアルタイム予測）。
  - **ベター（当日採用）**：まーぴーが**数値AIの出力CSV（担当別の翌月予測成約数）を読む**。
- 詳細は 仕様設計書 §13（推論エンドポイント）／ `../03_生成AI_まーぴー/要件定義書.md §7.3` を参照。

## 7. 機能要件（FR）

| ID | 要件 | 詳細 |
|---|---|---|
| FR-01 | データプレビュー | ダミーCSV（担当×月の時系列：活動量と成約数）を表で表示（入力の形を見せる。Step1） |
| FR-02 | 学習の穴埋め（fit） | 編集セルで `model.fit(___, ___)` を書く → [学習ボタン]で**その場で学習**（数秒）。Step2 |
| FR-03 | 推論の穴埋め（predict） | **営業担当を選び**、編集セルで `model.predict(___)` を書く → [推論ボタン]で**その担当の翌月予測成約数を表示**。Step3 |
| FR-04 | パラメータ調整 | `n_estimators` / `max_depth` を**スライダー**で変えて**再学習**できる。Step4 |
| FR-05 | 過学習の可視化 | train/val のスコアを**対比表示**（検証は時間分割＝未来の月）。パラメータを上げると訓練↑・検証↓を見せる（スライド54） |
| FR-06 | 特徴量重要度 | 学習後、どの活動（提案・面談・架電…）が翌月成約に効いたかを**簡易グラフ**で表示 |
| FR-07 | 編集範囲の明示（🔒／🟢） | 「やる場所＝🟢／やらない場所＝🔒」を画面冒頭一覧表とセル内コメントで明確に区切る |
| FR-08 | エラーの画面表示 | 編集コードが不正でも落ちず、**例外を捕捉して画面表示**し、修正を促す |
| FR-09 | 演習用／正解用の2系統 | 編集セル初期値が**空欄（演習用）／記入済み（正解用）**の2系統を用意する |
| FR-10 | リセット | 編集内容・パラメータを初期へ戻せる（任意） |

### 7.1 参加者の編集対象（穴埋めは2箇所＋パラメータ／🟢ゾーン）

1. **`fit`**（学習）：`model.fit(___, ___)`
2. **`predict`**（推論）：`model.predict(___)`
3. **パラメータ**：`n_estimators` / `max_depth` をスライダーで変更（コード編集ではない）

→ それ以外（データ読込・スコア計算・グラフ描画・ボタン制御）は **🔒固定領域**。

## 8. 非機能要件（NFR）

| ID | 要件 | 詳細 |
|---|---|---|
| NFR-01 | 同時利用 | **5インスタンス並行起動**しても相互に影響しない（各URL独立） |
| NFR-02 | 状態分離 | 学習済みモデル・結果は `st.session_state` に保持（インスタンス内で完結、揮発で可） |
| NFR-03 | サーバ計算 | 学習・推論はすべてサーバ側。参加者PCのCPUを使わない |
| NFR-04 | APIキー不要 | 外部AI呼び出しなし。計算はローカルライブラリ（XGBoost） |
| NFR-05 | 学習速度 | `fit`→`predict` が**数秒**で完了する規模（数百行） |
| NFR-06 | 安全な実行 | 編集コードは**限定した名前空間**で実行。不要な組み込み・I/Oを渡さない |
| NFR-07 | 復旧性 | リロードで初期状態に戻ってよい（永続化は不要） |
| NFR-08 | データ秘匿 | ダミーデータのみ。機微データは扱わない |

## 9. スコープ外（やらないこと）

- 本番SFDC/TSR等の実データ利用（ダミーのみ）。
- 高度なモデル選択・ハイパーパラメータ自動探索（手動スライダーで体感する範囲に限定）。
- モデルの永続保存・デプロイ運用（学習はその場・揮発で可）。
- IDE・ローカル環境構築（ブラウザ完結）。
- 外部API・認証。

## 10. 受入基準（HANDOFF §5.6 準拠）

- [ ] ダミーCSVで `fit`→`predict` が**数秒**で回り、**担当の翌月予測成約数**が表示される
- [ ] `fit`/`predict` が**編集セルの穴埋め**で、書かないと動かない（＝書く体験になる）
- [ ] パラメータを上げると **train↑・val↓** になり、**過学習が train/val 対比で可視化**される（検証は時間分割）
- [ ] 重要度で「**架電（量）より提案・面談が効く**」が見える
- [ ] 計算はすべて**サーバ側**（参加者PCに負荷なし）。APIキー不要
- [ ] 編集してよい範囲（🔒／🟢）が画面上で明確
- [ ] 演習用（空欄）／正解用（記入済み）の2系統が揃っている
- [ ] 5インスタンス並行で安定動作
- [ ] スライド51（学習＝つくる/推論＝つかう）・53（Step1-4）・54（過学習）と内容が一致

## 11. 確定事項・未確定

### 11.1 確定（2026-06-19・ユーザー決定）

- **目的変数＝翌月の成約数**（フォーキャスト）。`Ad数` は廃止。
- **特徴量＝先行活動（問い合わせ・架電・面談・案件提案）の当月/1ヶ月前/2ヶ月前＝12列**。契約数は使わない（自明回避）。
- **データ＝営業担当×月の時系列パネル**（30担当×30ヶ月＝900行）。`make_dummy.py` で生成・同梱。
- **モデル＝XGBoost**（スライド50準拠）。**過学習の見せ方＝train/val対比のみ**（学習曲線は描かない）。
- **検証＝時間分割**（過去で学習→未来で検証）。
- **作り方**：演習用／正解用の2系統＋🔒/🟢表記（ITパート共通）。

### 11.2 未確定・要確認

1. **特徴量・数値の現実性**：いまはダミー。本番SFDCの活動定義・粒度・成約のリードタイムをTech0/現場と擦り合わせ（スライド受領待ち）。
2. **リードタイム**：当日は「翌月（+1ヶ月）」。M&Aの長い営業サイクルを反映するなら +3ヶ月等への変更余地。
3. **連結（まーぴー）**：CSV（担当別の翌月予測）を確実に。エンドポイントは余力次第。
