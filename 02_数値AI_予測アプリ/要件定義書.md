> **実行方式（重要）**：本パートは **Community Cloud にデプロイして使う**（学習・推論の計算が役員PCに重いため）。参加者は配布URLを開き、**アプリ内の編集セルに fit/predict を書き、ボタンでサーバ側 exec**。IT編（ローカルで `app.py` 直接編集）とは方式が異なる。
> v4 の他の変更：データを **75担当×12ヶ月=900行**（月1〜12）に。連結は**CSV方式で実装確定**。趣旨（先行活動から翌月成約を予測／契約数は使わない＝自明回避）は維持。

## 1. 背景・位置づけ

- 90分ハンズオンの **第2パート＝数値AI（35分）**。**数字→数字**を予測する。
- 通しテーマ②「**AIがデータからルールを学ぶ**」。IT①「人が if を書く」との対比が肝。
- 体験のねらい：①AIはデータから学ぶ ②入力→処理→出力はどのAIも同じ ③改善は反復。

### 1.1 一本の糸

本パートは中核（学習fit→予測predict）。出力＝予測は `予測結果.csv` に保存され③まーぴーが話す。共通構造は [`../../00_共通/ハンズオン通しテーマ_一枚絵.md`](../../00_共通/ハンズオン通しテーマ_一枚絵.md)。

---

## 2. 目的・ゴール

「**数字を入れたら数字が出る**」数値AIを、参加者が **アプリ内編集セルに fit/predict を書いて** 動かす。さらに**数字を変えて反復**し、**過学習**を体感する。

- 題材：営業の先行活動 → 翌月の成約数を予測（フォーキャスト）。
- 特徴量：先行活動（問い合わせ・架電・面談・案件提案）の当月・1ヶ月前・2ヶ月前。**契約数（結果系）は使わない**（自明回避）。
- モデル：XGBoost 既定。Step2で**線形回帰との2択**（モデル選び）。
- 副次：重要度を見ると「**架電（量）より、提案・面談が翌月の成約に効く**」という示唆。

## 4. 題材・データ（SFDCイメージのダミー時系列）

- ストーリー：営業の先行活動 → 翌月の成約数を予測。**営業担当 × 月の時系列パネル**。

### 4.1 生データのスキーマ

| カラム | 例 | 役割 |
|---|---|---|
| `営業担当` | 濵田隼人 | キー（実名。固定8名＋ランダム67名） |
| `月` | 1〜12 | 時間（暦の月） |
| `問い合わせ数`/`架電数`/`面談数`/`案件提案数` | … | 先行活動（架電はあえて効きにくい／提案が最も効く） |
| `成約数` | 5 | その月の成約（→翌月にズラして正解 y に） |

- 規模：**75担当 × 12ヶ月 ＝ 900行**。特徴量＝各活動の当月・1・2ヶ月前（12列）、答え＝翌月の成約数。

---

## 5. 当日の進め方

配布URLを開き、🟢の編集セルだけを触る。

| Step | 内容 | やること |
|---|---|---|
| Step1 | データを見る | 手がかり(X)＝活動／答え(y)＝翌月成約（🔒 見るだけ） |
| Step2 | モデルを選ぶ | 種類（木/直線）とパラメータを選ぶ（🟢） |
| Step3 | 学習する | 編集セルに `model.fit(X_train, y_train)` → 🎓 学習（🟢） |
| Step4 | 推論する | 担当を選び `pred = model.predict(X_new)` → 🔮 推論（🟢） |
| 反復 | 数字を変える | Step2の数字を上げて過学習を観察（🟢） |

- **検証は時間分割**：序盤の月で学習し、終盤の月（未来）で検証。

### 5.1 3パート連結（②→③まーぴー）

- 学習が通ると、全担当の翌月予測を `data/予測結果.csv` に保存。③まーぴーがこれを読んで音声で答える。**CSV方式・実装済み**。
- 環境差（AI=Cloud／まーぴー=ローカル）の運用は デプロイ運用メモ §3。エンドポイント直呼びは将来。

## 7. 機能要件（FR）

| ID | 要件 | 詳細 |
|---|---|---|
| FR-01 | データプレビュー | 担当×月の時系列を表示。手がかり(X)／答え(y)を明示（Step1） |
| FR-02 | モデル選び（Step2） | `st.radio` で **木/直線** を選ぶ＋木はパラメータ slider |
| FR-03 | 学習の編集セル（fit） | 編集セルで `model.fit(___, ___)` を書く → [学習] で**サーバ側で学習**（数秒） |
| FR-04 | 推論の編集セル（predict） | 担当を選び `model.predict(___)` を書く → [推論] で**その担当の翌月予測成約数を表示** |
| FR-05 | 過学習の可視化 | train/val スコアを対比表示（時間分割）。数字を上げると訓練↑・検証↓ |
| FR-06 | 特徴量重要度 | 木モデルのとき、どの活動が効いたかを棒グラフ |
| FR-07 | 編集範囲の明示（🔒/🟢） | 画面の一覧表とセル内コメントで明確化 |
| FR-08 | エラーの画面表示 | 編集コードが不正でも落ちず、例外を捕捉して画面表示 |
| FR-09 | 演習用／正解用の2系統 | 編集セル初期値が**空欄（演習用）／記入済み（正解用）**の2系統 |
| FR-10 | 連結CSV出力 | 学習が通ると全担当の翌月予測を `data/予測結果.csv` に保存（③連結） |

### 7.1 参加者の編集対象（🟢）

1. **`fit`**：`model.fit(___, ___)`（編集セル）
2. **`predict`**：`pred = model.predict(___)`（編集セル）
3. モデル種類(radio)・パラメータ(slider)・担当(selectbox)

→ それ以外（データ・特徴量・時間分割・スコア・グラフ・CSV保存）は 🔒。

## 8. 非機能要件（NFR）

| ID | 要件 | 詳細 |
|---|---|---|
| NFR-01 | 同時利用 | 1人1URL（計5）並行で相互に影響しない |
| NFR-02 | サーバ計算 | 学習・推論はすべてサーバ側（Cloud）。役員PCのCPUを使わない |
| NFR-03 | APIキー不要 | 外部AI呼び出しなし |
| NFR-04 | 学習速度 | `fit`→`predict` が数秒（900行） |
| NFR-05 | 安全な実行 | 編集コードは限定名前空間で `exec`。不要な組み込み・I/Oを渡さない |
| NFR-06 | データ秘匿 | ダミーデータのみ |

## 10. 受入基準

- [ ] 配布URLで編集セルに `fit`→`predict` を書くと数秒で回り、担当の翌月予測成約数が出る
- [ ] `fit`/`predict` が空欄だとエラー表示で、書くと動く（書く体験／落とさない）
- [ ] Step2の数字を上げると train↑/val↓＝過学習が対比で見える（時間分割）
- [ ] 重要度（木モデル）で「架電より提案・面談が効く」が見える
- [ ] 学習が通ると `data/予測結果.csv` が生成され、③まーぴーが読める
- [ ] 演習用（空欄）／正解用（記入済み）の2系統が揃い、正解版はそのまま動く
- [ ] 5インスタンス並行で安定動作

## 11. 確定事項・未確定

### 11.1 確定

- **実行＝Community Cloud デプロイ。編集セル(text_area)＋サーバ側 exec**（計算が役員PCに重いため）。
- 目的変数＝翌月の成約数。特徴量＝先行活動の当月/1/2ヶ月前＝12列（契約数は使わない）。
- データ＝75担当×12ヶ月＝900行（月1〜12）。
- モデル＝XGBoost 既定＋線形回帰の2択。過学習＝train/val対比。検証＝時間分割。
- 連結＝CSV方式（実装済み）。詳細 [[jmc-it-app-decisions]]・[[jmc-handson-learning-design]]。

### 11.2 未確定・要確認

1. 特徴量・数値の現実性（本番SFDCの活動定義・粒度・リードタイムをTech0/現場と擦り合わせ）。
2. リードタイム：当日は翌月（+1ヶ月）。長い営業サイクルなら +3ヶ月等。
3. 連結CSVの環境差運用（AI=Cloud／まーぴー=ローカル）：当日は同梱サンプル or 講師が書き出して配置（デプロイ運用メモ §3）。
